# Bug-003: 模块构建模板接口不支持 module_type 数字值

| 属性 | 值 |
|------|------|
| **优先级** | P2 |
| **模块** | 模块管理 |
| **发现日期** | 2026-04-14 |
| **状态** | 未修复 |

## 复现步骤

1. 调用 `GET /api/modules/build-template/0`（0 表示独立仓库/JAR 类型）

## 期望结果

返回对应的构建命令模板

## 实际结果

返回 500 错误：`不支持的模块类型: 0`

## 根因分析

`getBuildTemplate` 方法使用字符串比较（如 `"JAR"`、`"VUE"`），但前端传入的是数字 `0`、`1`、`2`。类型不匹配导致无法识别。

## 修复建议

将 build-template 接口改为支持数字类型，或在 Service 层做数字到字符串的映射转换：
```java
0 -> "JAR" / "JAVA"
1 -> "MONOREPO"  
2 -> "MAVEN_MULTI"
```
