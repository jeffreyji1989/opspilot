# Bug-002: t_server 表缺少 jdk_versions / node_versions 列

| 属性 | 值 |
|------|------|
| **优先级** | P0（阻塞性） |
| **模块** | 服务器管理 / 仪表盘 |
| **发现日期** | 2026-04-14 |
| **状态** | 未修复 |

## 复现步骤

1. 登录后调用以下任意接口：
   - `GET /api/servers`
   - `GET /api/servers/by-env`
   - `GET /api/servers/{id}`
   - `GET /api/dashboard/stats`

2. 观察返回结果

## 期望结果

返回服务器列表或仪表盘统计数据

## 实际结果

返回 500 错误：
```
Unknown column 'jdk_versions' in 'field list'
```

SQL 尝试查询的缺失列：
- `jdk_versions`
- `node_versions`

## 根因分析

Server Entity 中定义了 `jdkVersions` 和 `nodeVersions` 字段（MyBatis Plus 自动映射为 `jdk_versions` 和 `node_versions`），但数据库 `t_server` 表中没有这两列。

## 影响范围

- 服务器列表页面完全无法加载
- 按环境分组查询不可用
- 仪表盘统计（依赖服务器数据）不可用
- 服务器详情页不可用

## 修复建议

```sql
ALTER TABLE t_server ADD COLUMN jdk_versions VARCHAR(256) DEFAULT '';
ALTER TABLE t_server ADD COLUMN node_versions VARCHAR(256) DEFAULT '';
```
