# BUG-001: 项目删除未校验关联模块/服务

| 字段 | 内容 |
|------|------|
| **编号** | BUG-001 |
| **严重程度** | P0 |
| **模块** | 项目管理 |
| **状态** | ✅ Fixed & Verified |
| **发现日期** | 2026-04-12 |
| **修复日期** | 2026-04-12 |
| **发现人** | QA Engineer |
| **修复人** | Full-Stack Developer |

## Bug 描述

删除项目时未检查该项目下是否有关联的模块（module）或服务实例（service_instance），直接执行逻辑删除。这会导致关联数据变为孤立记录，破坏数据一致性。

## 复现步骤

1. 调用 `POST /api/projects` 创建项目
2. 调用 `POST /api/projects/{projectId}/modules` 为该项目添加模块
3. 调用 `DELETE /api/projects/{projectId}` 删除项目

## 期望结果

后端返回 400 错误，错误码 30xxx，提示信息：「项目下存在模块/服务，不允许删除」

## 实际结果（修复前）

```json
{"code":200,"message":"success","data":null}
```

项目被成功逻辑删除（`deleted=1`），但关联的模块记录仍然存在，`project_id` 指向已删除的项目。

## 根因分析

`ProjectController.java` 第 56-64 行：
```java
@DeleteMapping("/{id}")
public Result<Void> delete(@PathVariable Long id, HttpServletRequest request) {
    Project project = projectService.getById(id);
    projectService.removeById(id);  // ← 直接删除，无关联校验
    ...
}
```

## 修复方案

### 1. ProjectService 接口新增方法

**文件**: `ProjectService.java`
```java
/** 删除项目前校验关联模块和服务实例 */
void deleteProject(Long id);
```

### 2. ProjectServiceImpl 实现删除前校验

**文件**: `ProjectServiceImpl.java`
```java
@Override
@Transactional
public void deleteProject(Long id) {
    // 检查关联模块
    long moduleCount = moduleMapper.selectCount(
            new LambdaQueryWrapper<Module>().eq(Module::getProjectId, id));
    if (moduleCount > 0) {
        throw new BusinessException(30002, "项目下存在模块，不允许删除");
    }
    // 检查关联服务实例（通过 module 关联）
    List<Module> modules = moduleMapper.selectList(
            new LambdaQueryWrapper<Module>().eq(Module::getProjectId, id));
    if (!modules.isEmpty()) {
        List<Long> moduleIds = modules.stream().map(Module::getId).toList();
        long serviceCount = serviceInstanceMapper.selectCount(
                new LambdaQueryWrapper<ServiceInstance>().in(ServiceInstance::getModuleId, moduleIds));
        if (serviceCount > 0) {
            throw new BusinessException(30003, "项目下存在服务实例，不允许删除");
        }
    }
    this.removeById(id);
}
```

### 3. ProjectController 调用新方法

**文件**: `ProjectController.java`
```java
@DeleteMapping("/{id}")
public Result<Void> delete(@PathVariable Long id, HttpServletRequest request) {
    Project project = projectService.getById(id);
    projectService.deleteProject(id);  // ← 替换 removeById，包含关联校验
    ...
}
```

# BUG-002: 项目编码重复暴露原始 SQL 异常信息

| 字段 | 内容 |
|------|------|
| **编号** | BUG-002 |
| **严重程度** | P1 |
| **模块** | 项目管理 |
| **状态** | ✅ Fixed & Verified |
| **发现日期** | 2026-04-12 |
| **修复日期** | 2026-04-12 |

## Bug 描述

创建项目时 project_code 重复，直接暴露了 SQL 异常堆栈。

## 修复方案

在 `GlobalExceptionHandler.java` 中添加 `DuplicateKeyException` 处理器：

```java
@ResponseStatus(HttpStatus.CONFLICT)
@ExceptionHandler(DuplicateKeyException.class)
public Result<?> handleDuplicateKeyException(DuplicateKeyException e) {
    log.warn("Duplicate key exception: {}", e.getMessage());
    return Result.error(10002, "项目编码已存在");
}
```

## 验证结果

```bash
# 创建重复 project_code 的项目
$ curl -X POST http://localhost:8080/api/projects \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer <token>" \
    -d '{"projectCode":"dup-test-002","projectName":"去重测试2","ownerId":1}'
# 返回 HTTP 409：
{
    "code": 10002,
    "message": "项目编码已存在",
    "data": null
}
```

✅ **验证通过**：返回 HTTP 409 + 友好提示「项目编码已存在」，不再暴露 SQL 异常堆栈。

# BUG-003: 服务器创建接口 NPE

| 字段 | 内容 |
|------|------|
| **编号** | BUG-003 |
| **严重程度** | P1 |
| **模块** | 服务器管理 |
| **状态** | ✅ Fixed & Verified |
| **发现日期** | 2026-04-12 |
| **修复日期** | 2026-04-12 |

## Bug 描述

请求参数为空时直接 NPE，没有参数校验。

## 修复方案

### 1. ServerCreateRequest 添加校验注解

**文件**: `ServerController.java`
```java
@Data
public static class ServerCreateRequest {
    @NotNull(message = "服务器信息不能为空")
    private Server server;
    private String sshPassword;
}
```

### 2. create 方法添加 @Valid 注解

```java
@PostMapping
public Result<Server> create(@Valid @RequestBody ServerCreateRequest req, HttpServletRequest request) {
```

### 3. GlobalExceptionHandler 添加 HTTP 400 状态码

```java
@ResponseStatus(HttpStatus.BAD_REQUEST)
@ExceptionHandler(MethodArgumentNotValidException.class)
public Result<?> handleValidationException(MethodArgumentNotValidException e) {
    String msg = e.getBindingResult().getFieldErrors().stream()
            .map(fe -> fe.getField() + ": " + fe.getDefaultMessage())
            .findFirst()
            .orElse("参数校验失败");
    return Result.error(10001, msg);
}
```
