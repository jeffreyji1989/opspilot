# OpsPilot UI 设计规范

> **文档版本**: v1.0  
> **创建日期**: 2026-04-14  
> **作者**: UI/UE 设计师  
> **适用项目**: OpsPilot 运维管理系统 V1.0

---

## 1. 设计原则

1. **效率优先** — 运维操作以高频、快捷为核心，减少点击层级，关键操作一步到位
2. **信息密度适中** — 后台管理系统需要较高信息密度，但不可拥挤；数据表格以清晰可读为第一目标
3. **安全感知** — 生产环境操作必须有明确的视觉警示，与开发/测试环境形成强区分
4. **一致性** — 所有页面遵循统一的设计 Token（配色、字体、间距、圆角）

## 5. 响应式适配方案

### 5.1 1920px（大屏桌面）

- 侧边栏: 展开 240px
- 内容区最大宽度: 1400px，居中
- 统计卡片: 4 列等宽
- 数据表格: 显示全部列
- 日志区域: 全屏宽度（内容区）

### 5.2 1440px（标准桌面）

- 侧边栏: 展开 240px
- 内容区最大宽度: 1200px，居中
- 统计卡片: 4 列等宽
- 数据表格: 显示全部列
- 日志区域: 全屏宽度（内容区）

### 5.3 768px（平板）

- 侧边栏: 折叠为图标模式 64px，或完全隐藏（汉堡菜单）
- 内容区: 无最大宽度限制，全屏
- 统计卡片: 2 列等宽
- 数据表格: 隐藏次要列（创建时间、负责人、描述等），保留核心列
- 表单: 双列 → 单列
- 对话框: 宽度 90%，最大 480px
- 操作按钮组: 垂直排列或收起至「更多」下拉

## 6. 页面状态覆盖

每个页面必须覆盖以下 5 种状态：

| 状态 | 说明 | 实现方式 |
|------|------|----------|
| **正常** | 数据加载成功，展示完整内容 | Element Plus 标准组件 + 自定义样式 |
| **Loading** | 数据请求中 | `v-loading` 指令 + 骨架屏（首屏） |
| **空数据** | 列表/表格无数据 | `el-empty` 组件 + 引导操作按钮 |
| **错误** | 请求失败 / 网络异常 | `el-alert` 错误态 + 重试按钮 |
| **异常流程** | 权限不足 / 操作冲突 | 自定义错误页面 + 返回首页按钮 |

## 7. 动效规范

| 动效 | 时长 | 缓动函数 | 用途 |
|------|------|----------|------|
| 淡入 | 200ms | `ease-out` | 页面切换、弹窗出现 |
| 淡出 | 150ms | `ease-in` | 弹窗关闭 |
| 滑入（侧边栏） | 300ms | `ease-out` | 侧边栏展开/折叠 |
| 缩放（按钮点击） | 100ms | `ease-out` | 按钮按压缩放 0.98 |
| 脉冲（部署中状态点） | 1.5s | `ease-in-out infinite` | 在线/部署中状态指示 |
| 进度条过渡 | 500ms | `linear` | 部署进度条 |
| 行悬浮 | 150ms | `ease-out` | 表格行悬浮变色 |
