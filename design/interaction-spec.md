## 1. 全局交互规范

### 1.1 二次确认机制

所有涉及**生产环境**或**数据删除**的操作，必须弹出二次确认对话框。

#### 1.1.1 触发场景

| 操作 | 触发条件 | 对话框标题 |
|------|----------|-----------|
| 生产环境发版 | 服务 env = prod | ⚠️ 生产环境发版确认 |
| 生产环境重启 | 服务 env = prod | ⚠️ 生产环境重启确认 |
| 生产环境回退 | 服务 env = prod | ⚠️ 生产环境回退确认 |
| 删除服务/服务器 | 任何删除操作 | 确认删除 |
| 添加生产服务器 | env = prod | ⚠️ 生产环境操作确认 |

#### 1.1.2 对话框设计规范

```
┌──────────────────────────────────────────┐
│ ⚠️ 生产环境操作确认              [ × ]   │
├──────────────────────────────────────────┤
│                                          │
│ 操作类型: 发版部署                        │
│ 服务名称: order-service-prod              │
│ 目标分支: main                            │
│ 目标版本: v1.0.2                          │
│                                          │
│ 此操作将影响生产环境服务，请确认已做好     │
│ 充分测试并选择了正确的目标版本。           │
│                                          │
│              [ 取消 ]  [ ⚠️ 我已确认，执行操作 ]  │
└──────────────────────────────────────────┘
```

- **关闭按钮**: 右上角 × + 点击遮罩层关闭（生产环境操作关闭遮罩 = 取消）
- **取消按钮**: 默认按钮（Default 样式），点击关闭对话框
- **确认按钮**: Danger 样式，带 ⚠️ 图标，文字明确说明后果
- **生产环境操作**：确认按钮使用 `type="danger"`，文字为「我确认，执行操作」
- **非生产删除操作**：确认按钮使用 `type="danger"`，文字为「确认删除」

### 1.2 操作进度展示

所有异步操作（发版、重启、探测等）通过以下方式展示进度：

#### 1.2.1 发版部署进度

```
步骤指示器（Steps 组件）+ 实时日志区域

Step 1: 拉取代码     ████████████ 完成 (5s)
Step 2: 编译构建     ████████████ 完成 (1m 20s)
Step 3: 打包产物     ████████░░░░ 进行中...
Step 4: 上传至服务器  ░░░░░░░░░░░░ 等待中
Step 5: 切换版本     ░░░░░░░░░░░░ 等待中
Step 6: 重启服务     ░░░░░░░░░░░░ 等待中
Step 7: 健康检查     ░░░░░░░░░░░░ 等待中

┌────────────────────────────────────────┐
│ [实时日志]                              │
│ [10:05:01] 开始拉取代码...              │
│ [10:05:03] git fetch origin main ✓     │
│ [10:05:04] git checkout main ✓         │
│ [10:05:05] 拉取代码完成 ✓ (5s)         │
│ [10:05:06] 开始编译构建...             │
└────────────────────────────────────────┘
```

**交互要点**:
- 弹窗宽度 680px，不可拖动大小
- 部署进行中时，「取消」按钮仅非生产环境可用
- 失败时：停止在失败步骤，红色高亮，展示错误日志，提供「一键回退」按钮
- 完成后：自动跳转到结果页面（Step 4）

#### 1.2.2 服务器探测进度

```
添加服务器 → 填写表单 → 点击「连接并探测」
  ↓
按钮显示 loading 状态，文字变为「正在连接并探测...」
  ↓
弹出探测结果对话框，展示 OS/JDK/Node/Docker 等信息
  ↓
用户确认 → 保存成功
```

### 1.3 错误处理

| 场景 | 处理方式 | 用户操作 |
|------|----------|----------|
| 页面数据加载失败 | `el-alert` 错误态 + 重试按钮 | 点击重试 |
| 列表为空 | `el-empty` 组件 + 引导操作按钮 | 点击引导按钮 |
| 部署失败 | 弹窗展示失败步骤和错误日志 + 「一键回退」 | 查看详情 / 回退 |
| SSH 连接失败 | Toast 错误提示 + 「重新连接」按钮 | 重新连接 |
| 健康检查失败 | 展示最近日志片段 + 提示可能原因 | 查看日志 / 手动排查 |
| 网络超时 | Toast 超时提示 + 「重试」按钮 | 重试 |
| 并发冲突 | Toast 提示「该服务正在部署中，请稍后再试」 | 等待后重试 |
| 权限不足 | 全屏 403 页面 + 返回首页按钮 | 返回首页 |

#### 1.3.1 Toast 通知规范

| 类型 | 颜色 | 持续时间 | 关闭方式 |
|------|------|----------|----------|
| 成功 (success) | 绿色 `#10B981` | 3 秒 | 自动关闭 + 手动 × |
| 警告 (warning) | 橙色 `#F59E0B` | 5 秒 | 自动关闭 + 手动 × |
| 错误 (error) | 红色 `#EF4444` | 5 秒 | 手动 × |
| 信息 (info) | 蓝色 `#3B82F6` | 3 秒 | 自动关闭 + 手动 × |

#### 1.3.2 空数据状态规范

```
┌──────────────────────────────┐
│                              │
│          📁                  │  ← 大图标（64px）
│                              │
│      暂无项目                │  ← 标题（16px / 600）
│  还没有创建任何项目...        │  ← 描述（13px / 400）
│                              │
│      [ ➕ 新建项目 ]          │  ← 引导操作按钮
│                              │
└──────────────────────────────┘
```

- 垂直居中展示
- 图标使用 emoji 或 SVG 占位图
- 描述文字说明当前状态和下一步操作
- 引导按钮使用 Primary 样式

## 2. 页面级交互说明

### 2.1 登录页

#### 状态流转

```
[打开登录页]
  ↓
正常状态：显示登录表单
  ↓
输入用户名/密码 → 点击登录
  ↓
按钮 loading 状态（禁用重复提交）
  ↓
成功 → Toast「登录成功，正在跳转...」→ 跳转 Dashboard
失败 → Toast「用户名或密码错误，请重试」→ 清空密码框
网络异常 → Alert「网络连接超时，请检查网络后重试」
```

#### 交互细节
- **回车提交**：密码框聚焦时按 Enter 触发登录
- **记住我**：勾选后 7 天内免登录（localStorage 存储 token）
- **密码可见**：点击眼睛图标切换密码可见性
- **表单验证**：用户名和密码为必填，失焦时即时校验

### 2.2 Dashboard 仪表盘

#### 数据加载
- 页面打开时自动加载统计卡片、环境分布、服务健康、最近操作
- 各模块独立 loading，互不影响
- 单个模块加载失败不影响其他模块展示

#### 统计卡片交互
- 鼠标悬浮时卡片轻微上浮（box-shadow 增强）
- 点击卡片跳转到对应管理页面

#### 服务健康交互
- 运行中服务：绿色圆点 + 脉冲动画
- 异常服务：红色圆点，点击跳转到服务详情页
- 部署中服务：蓝色脉冲圆点

#### 最近操作交互
- 最多展示 10 条，超出部分通过「查看全部」跳转到操作日志页
- 每条记录可点击，跳转到对应的服务详情/操作详情

### 2.3 项目管理页

#### 搜索与筛选
- 搜索条件：项目名称（模糊匹配）、负责人、标签
- 点击「搜索」按钮触发查询
- 按 Enter 键触发搜索
- 「重置」按钮清空所有搜索条件并重新加载

#### 表格交互
- **项目名称**：点击名称可编辑
- **标签**：点击标签可筛选同标签项目
- **模块数**：点击跳转到模块管理对话框
- **编辑**：打开编辑对话框，回填数据
- **删除**：二次确认 → 校验是否有关联模块 → 执行删除
  - 有模块时：删除按钮禁用，Tooltip 提示「请先删除所有模块」

#### 模块管理对话框
- 以对话框形式展示，宽度 900px
- 显示模块列表 + 添加模块按钮
- 删除模块时校验是否有关联服务

### 2.4 服务器管理页

#### 环境分组展示
- 按 dev → test → staging → prod 顺序排列
- 每组有环境标识色点和服务器数量
- 生产环境组：红色左边框 + ⚠️ 高风险标签

#### 服务器卡片交互
- **查看详情**：弹出服务器详情对话框，展示基本信息 + 环境探测结果 + 已部署服务列表
- **编辑**：打开编辑对话框（不可修改环境类型）
- **重新探测**：手动触发环境探测，按钮 loading 状态
- **删除**：有服务运行时禁止删除，Tooltip 提示「请先停止该服务器上的所有服务」

#### 添加服务器流程
```
点击「添加服务器」
  ↓
填写表单（主机名、IP、端口、用户名、密码、环境类型）
  ↓
环境类型选择 prod → 弹出二次确认对话框
  ↓
确认后点击「连接并探测」
  ↓
按钮 loading → SSH 连接 → 自动部署公钥 → 环境探测
  ↓
探测完成 → 展示探测结果 → 用户确认 → 保存成功
```

#### 添加生产服务器二次确认
```
┌──────────────────────────────────────┐
│ ⚠️ 生产环境操作确认           [ × ]  │
├──────────────────────────────────────┤
│                                      │
│ 您正在添加生产环境服务器              │
│                                      │
│ 主机名: prod-order-01                │
│ IP 地址: 192.168.1.100              │
│ 环境: 🔴 生产（prod）               │
│                                      │
│ 生产环境操作具有高风险，请确认信息无  │
│ 误。                                 │
│                                      │
│   [ 取消 ]  [ ⚠️ 我确认，继续添加 ]  │
└──────────────────────────────────────┘
```

### 2.5 服务管理页

#### 列表视图
- 筛选条件：项目名称、环境类型、运行状态
- 运行状态筛选支持多选
- 表格操作列：详情 / 发版 / 重启 / 删除

#### 服务详情视图
- 点击服务名称或「详情」按钮进入
- 顶部信息栏：服务名称、所属项目/模块、运行状态、环境、服务器、端口、当前版本
- 操作按钮组：发版 / 重启 / 回退 / 配置 / 定时 / 删除
- Tab 页签：发版记录 / 实时日志 / 配置管理 / 服务监控 / 定时发版

#### Tab 1: 发版记录
- 列表展示历史发版记录
- 每行操作：查看详情 / 回退到此版本
- 回退操作需要二次确认

#### Tab 2: 实时日志
- **实时/历史切换**：Radio 切换
- **级别过滤**：全部 / ERROR / WARN / INFO / DEBUG
- **关键字搜索**：输入关键字后实时高亮匹配行
- **自动滚动**：默认开启，可暂停/恢复
- **清空**：清空当前展示区域（不影响服务器日志）
- **下载**：下载当前日志为 .log 文件

**日志颜色**:
- ERROR 行：红色 `#F87171`
- WARN 行：橙色 `#FBBF24`
- INFO 行：蓝色 `#60A5FA`
- 其他：灰色 `#D4D4D4`

#### Tab 3: 配置管理
- 环境变量：Key-Value 表格，可添加/编辑/删除
- JVM 参数：文本域编辑
- 启动脚本：代码编辑器展示（只读，可导出）
- 保存提示：「保存（下次重启生效）」

#### Tab 4: 服务监控
- 监控卡片网格布局（3 列）
- 手动刷新按钮 + 自动刷新开关（默认 30s 间隔）
- 响应时间趋势图（预留 V2）

#### Tab 5: 定时发版
- 该服务的定时任务列表
- 操作：编辑 / 立即执行 / 删除

#### 服务操作交互

**重启**:
```
点击「🔄 重启」
  ↓
生产环境 → 二次确认对话框
非生产环境 → 直接执行
  ↓
弹窗进度展示：停止 → 等待 → 启动 → 健康检查
  ↓
完成 → Toast 通知
```

**回退**:
```
点击「⏪ 回退」
  ↓
弹出历史版本列表（最近 10 个）
  ↓
选择目标版本
  ↓
生产环境 → 二次确认对话框
  ↓
执行回退：切换软链 → 重启 → 健康检查
  ↓
完成 → Toast 通知 + 记录操作日志
```

**删除**:
```
点击「🗑️ 删除」
  ↓
校验服务状态 = 已停止（否则禁止删除）
  ↓
二次确认对话框
  ↓
可选：是否同时删除服务器上的部署文件
  ↓
执行删除（软删除）
```

### 2.6 发版部署页（4 步向导）

#### Step 1: 选择版本
- 三种选择方式：分支 / Tag / Commit ID
- 分支/Tag 下拉列表展示远程拉取的列表
- Commit ID 输入框支持手动输入
- 下方展示最近 5 次提交记录（Commit message、作者、时间）

#### Step 2: 确认信息
- 展示服务名称、环境、当前版本、目标版本
- 生产环境显示预计影响说明
- 生产环境弹出二次确认对话框

#### Step 3: 执行部署
- 7 步进度条实时展示
- 实时日志区域可展开/收起
- 非生产环境支持「取消部署」
- 失败时自动停止，展示错误

#### Step 4: 部署结果
- 成功：绿色 ✅，显示总耗时和新版本号
- 失败：红色 ❌，显示失败步骤和错误，提供「一键回退」
- 记录到操作日志

### 2.7 Git 凭证管理页

#### 列表交互
- 认证名称：展示 🔑 图标 + 名称
- 认证类型：Tag 标签区分 SSH Key / HTTPS Token
- 指纹/状态：敏感信息脱敏显示（SHA256:abc***xyz）
- 关联模块数 > 0 时禁止删除

#### 添加/编辑
- SSH Key 模式：粘贴私钥内容（多行文本域）+ 可选私钥密码
- HTTPS Token 模式：粘贴 Token 值 + 可选适用域名
- 编辑时私钥/Token 不回填，需要重新输入或选择「保留原值」

### 2.8 操作日志页

#### 多条件筛选
- 操作人（输入框）
- 操作类型（下拉多选）
- 环境（下拉多选）
- 操作结果（下拉多选）
- 时间范围（日期范围选择器）
- 操作对象（输入框）
- 支持组合搜索

#### 表格交互
- 操作类型列：带图标标识（🚀 发版 / 🔄 重启 / ⏪ 回退等）
- 环境列：彩色 Tag 区分
- 结果列：彩色 Tag 区分
- 操作列：详情按钮

#### 日志详情
- 对话框展示完整操作信息
- 包含操作 ID、操作人、类型、对象、环境、详情、结果、耗时、IP、时间

#### 导出
- 点击「导出 CSV」按钮
- 导出当前筛选结果
- 文件名格式：`opspilot-operation-log-{日期}.csv`

### 2.9 定时部署任务页

#### 列表交互
- 开关控制任务启用/暂停
- 立即执行：二次确认后加入执行队列
- 编辑：回填表单数据
- 删除：二次确认

#### 创建/编辑
- 基础信息：任务名称、目标服务、目标分支
- 触发方式：单次（日期时间选择器）/ Cron（文本输入 + 下次执行时间预览）
- 高级选项：
  - 失败回退策略：不回退 / 自动回退到上一版本 / 健康检查失败自动回退
  - 钉钉通知：开关 + 通知场景（成功/失败）+ @通知人

#### Cron 表达式帮助
```
┌────────────────────────────────────────────┐
│ 💡 Cron 表达式示例                          │
│                                            │
│ 0 2 * * *     每天凌晨 2 点                │
│ 0 */6 * * *   每 6 小时执行一次            │
│ 0 10 * * 1    每周一上午 10 点             │
│ 0 0 1 * *     每月 1 日零点                │
│                                            │
│ 格式: 分 时 日 月 星期                      │
└────────────────────────────────────────────┘
```

## 4. 响应式布局方案

## 5. 动效规范

| 动效名称 | 时长 | 缓动函数 | 触发场景 |
|----------|------|----------|----------|
| 淡入 | 200ms | `ease-out` | 页面切换、弹窗出现、组件加载完成 |
| 淡出 | 150ms | `ease-in` | 弹窗关闭、Toast 自动消失 |
| 滑入（侧边栏） | 300ms | `ease-out` | 侧边栏展开/折叠 |
| 缩放（按钮点击） | 100ms | `ease-out` | 按钮按压缩放至 0.98 |
| 脉冲（状态点） | 1.5s | `ease-in-out infinite` | 部署中/探测中状态指示点 |
| 进度条过渡 | 500ms | `linear` | 部署进度条增长 |
| 行悬浮 | 150ms | `ease-out` | 表格行悬浮背景色变化 |
| 卡片悬浮 | 150ms | `ease-out` | 卡片 box-shadow 增强 + translateY(-2px) |

## 7. 无障碍（Accessibility）

- 所有交互元素支持键盘 Tab 导航
- 按钮和链接有明确的 `aria-label`
- 颜色对比度满足 WCAG AA 标准（≥ 4.5:1）
- 表单字段有 `label` 关联
- 错误提示同时使用颜色和文字说明
- 图片/图标有 `alt` 属性
